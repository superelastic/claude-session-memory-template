# Reference Documentation

This directory contains reusable reference material, methodologies, and gotchas.

## What Belongs Here

- **Methodologies** - Repeatable processes (e.g., validation workflows)
- **Gotchas** - Non-obvious issues and their solutions
- **Quick references** - Cheat sheets, parameter guides
- **External learnings** - Lessons from external sources applied to this project

## What Does NOT Belong Here

- **Investigations** - Go in `docs/investigations/` (hypothesis-driven research)
- **Architecture decisions** - Go in `docs/decisions/` (ADRs)
- **API documentation** - Usually lives in code or dedicated API docs

## Naming Convention

Use descriptive lowercase names with underscores: `validation_methodology.md`, `thetadata_gotchas.md`
