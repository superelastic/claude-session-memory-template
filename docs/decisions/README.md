# Architecture Decisions

This directory contains Architecture Decision Records (ADRs) documenting significant technical decisions.

## When to Create a Decision Doc

- Choosing between architectural approaches
- Selecting libraries or frameworks
- Defining system boundaries or interfaces
- Making trade-offs with long-term implications

## Format

Use a simple structure:

```markdown
# Decision: [Title]

## Context
What situation prompted this decision?

## Options Considered
1. Option A - pros/cons
2. Option B - pros/cons

## Decision
What we chose and why.

## Consequences
What this decision means going forward.
```

## Naming Convention

`YYYY-MM-DD-brief-title.md` (e.g., `2025-01-15-use-oauth-pkce.md`)
